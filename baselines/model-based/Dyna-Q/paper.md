# Canonical Dyna-Q (Sutton) Overview

Dyna-Q combines **model-free temporal-difference learning** and **model-based planning** inside one loop. In Sutton and Barto's tabular formulation, the agent improves action values both from real interaction and from simulated experience generated by a learned model.

## Core Components

- **Action-value function** `Q(s, a)`: estimates expected discounted return for each state-action pair.
- **Behavior policy**: usually epsilon-greedy w.r.t. `Q` for exploration.
- **Model** `Model(s, a) -> (r, s')` (and terminal status in episodic tasks): stores/predicts one-step environment dynamics.
- **Planning budget** `n`: number of model-generated updates per real step.

## Canonical Update Equations

For each real transition `(s, a, r, s')`:

1. **Direct RL (Q-learning backup)**  
   `Q(s,a) <- Q(s,a) + alpha * [r + gamma * max_a' Q(s',a') - Q(s,a)]`  
   (for terminal `s'`, the bootstrap term is omitted).

2. **Model learning**  
   Store/update the observed transition in the model for `(s, a)`.

3. **Planning (repeat `n` times)**  
   - Sample previously seen `(s~, a~)`.
   - Query model: `(r~, s~') = Model(s~, a~)`.
   - Apply the same TD backup to `Q(s~, a~)`.

## Canonical Dyna-Q Loop

At each environment step:

1. Observe current state `S`.
2. Select action `A` using epsilon-greedy from `Q`.
3. Execute `A`, observe reward and next state.
4. Update `Q` from the real transition.
5. Update the model with that transition.
6. Run `n` planning updates from model samples.

This architecture improves sample efficiency: every expensive real step produces one direct update plus multiple cheaper planning updates.
